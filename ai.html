AI Integration

Use Listenara from local AI agents.

Listenara is the Mac app. The AI skill is an optional adapter that tells Codex, Claude Code, and similar local agents how to open and control the local preview.

What "run Listenara" means

In an AI workflow, "run Listenara" means the agent runs listenara preview --json, reads the returned previewURL, and opens that local URL inside the AI tool's own preview or browser when available.

The AI workflow should not open Safari or Chrome automatically unless you ask. It should not start recording automatically.

What you need

  • Install Listenara.app on your Mac.
  • Use an AI tool that can run local commands.
  • For the best experience, use an AI tool that can open localhost previews.

If the AI tool cannot open local previews, it should show the local URL so you can open it yourself.

Command contract

Agents should prefer listenara on PATH. If that is unavailable, they can use /Applications/Listenara.app/Contents/MacOS/listenara.

Inside the source repo, the development command is .build/release/listenara.

Privacy boundary

Launching Listenara, opening the local preview, and running local Listenara commands keeps transcription and translation local on your Mac. If you ask an AI assistant to read, summarize, rewrite, or transform transcript text, that text may enter that assistant's chat context.

Codex, Claude, and other agents

The shared Listenara skill works through the same local command contract. Codex and Claude Code use small adapter files, while other local agents can use the shared skill instructions if they support local shell commands and localhost previews.
